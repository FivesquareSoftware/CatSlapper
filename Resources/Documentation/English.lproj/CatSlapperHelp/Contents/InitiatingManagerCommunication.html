Initiating Communication With Tomcat Manager

NOTE: All uses of the Tomcat Manager application require that authentication for that instance be configured. CatSlapper does not currently do this for you. To learn more about configuring the Tomcat Manager application for authentication see the .

When a configuration is first added, and CatSlapper has yet to retrieve any component information from Tomcat Manager, you will see just the root element—the current instance of Tomcat. CatSlapper ships set to perform automatic updates of component information at regular intervals. However, you can initiate the communication manually if you wish.

To manually initiate the communication between CatSlapper and Tomcat Manager:

Click the component reload button in the lower right of the "Components" outline view, or click "Action" from the main menu and select "Update Component Info".

To change the frequency with which CatSlapper requests component updates from Tomcat Manager:

Click "CatSlapper" from the main menu and select "Preferences".

Select the "Manger" pane and enter a new value in seconds in the text field.

To turn off automatic requests for updates from Tomcat Manager:

Click "CatSlapper" from the main menu and select "Preferences".

Select the "Manger" pane and check "manually".
